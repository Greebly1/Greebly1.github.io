Top-Down Shooter | Fool's Gold

General Info

This game was made with Unity C#

I used this project to experiment with decoupling game systems extensively (I probably went too far).

The camera system is exceedingly robust, the player has extremely fine-tune control over it.

Multiple element types to spice things up (implemented with bit-masking)

Feature complete but not polished

-->Github Repo<--

Input

All player input is seperated into self-contained state machines. For example:

-User Interface Input

-Action/Ability Input

-Camera Input

Systems communicate using events in a decoupled manner. For instance, the UI select event signals the camera controller to attempt to focus on a gameobject

So multiple programmers can work on user input systems at a time without needing to know implementation details of other systems.

Also, designers can easily hook up input events to behaviors in game.

Locomotion

All pawns use root motion taken from Mixamo animations

Pawns leverage inheritance to extend the front-end of the default pawn.

For instance, a pawn supports movement commands, and a humanoid pawn additionally supports crouch commands

For aiming, item holding, and crouching I utilized Unity's procedural animation tools for inverse kinematics and animation layers.

Muzzle Flash

The effect is built with the Unity VFX graph

The outwards sparks use radial UVs, so we can randomize the amount of sparks by compressing the UV coordinates in the X with texture repeat enabled

The weapon, and FX textures were made by me

I also implemented basic procedural animations for the weapon aiming

Wave System

I created a data-driven tool for designers to author enemy waves

Designers create custom scriptable obects that encapsulate wave data

The wave manager in game, is a singleton that handles spawning the waves.

Wave manager and level manager communicate through static events in a decoupled manner
